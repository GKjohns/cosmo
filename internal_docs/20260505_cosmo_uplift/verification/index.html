Deviations

Deviations

Subscriptions scoped to organizations, not users. Margin: organization_id; Daylight: user_id. Cosmo follows Margin because Sprint 2's multi-tenancy is org-first — a user upgrades the active org's plan.
Generic resource map. items, ai_tokens, storage_bytes instead of Margin's notebook-specific resources. Other resources are commented placeholders inside 007_usage_tracking.sql for projects to extend.
profiles.test_tier column for the employee tier-switcher. Smallest-possible thing that lets getUserTier return a stub-mode override. No separate dev_tier_overrides table.
stripe server SDK is a cosmo dep; @stripe/stripe-js is not. Server SDK lazy-imports inside the live branch; client SDK is project-installs-on-flip. EmbeddedCheckout.vue falls back to the demo banner if the dep is missing.
Lean dev script. Cosmo ships concurrently --names nuxt,inngest. Projects that flip to live add stripe listen --forward-to localhost:3000/api/stripe/webhook themselves; recipe is documented in root CLAUDE.md.

Sprint 6 — Chat upgrade

Mirror nuxt-ui-templates/chat; cross-reference AIR-Bot's MessageContent.vue for the parts renderer. New 008_chats.sql drops unused ai_messages/ai_conversations; flatter chats table with messages jsonb. Old app/pages/app/ai.vue becomes a redirect.

Empty-state → uuid wiring (load-bearing)

  1. User submits in /app/chat. loading.value = true blocks suggestion buttons + submit.
  2. await $fetch('/api/chats', { method: 'POST', body: { id, message } }) — server inserts the chat row + first user message.
  3. refreshNuxtData('chats') — sidebar picks up the new chat.
  4. await navigateTo('/app/chat/<id>').
  5. [id].vue hydrates from useFetch('/api/chats/<id>'), builds the Chat with the persisted user message.
  6. onMounted sees messages.length === 1 && role === 'user' → calls chat.regenerate() to kick off the assistant turn.
  7. The shared [view-transition-name:chat-prompt] keeps the prompt locked across the navigation. Submit is disabled while loading so a double-tap can't create two chats.

Deviations

Empty state lives at /app/chat. Template puts it at /; cosmo's marketing layout owns /, so the chat empty state is /app/chat/index.vue.
jsonb persistence, no Drizzle. Template uses hub:db + Drizzle with a normalized messages table. Cosmo follows AIR-Bot: one row per thread, full UIMessage[] in messages jsonb. Simpler RLS, simpler hydration.
Id-diff persistence in onFinish. Re-fetches latest, builds existingIds, filters responseMessages to unseen — re-emitted parts don't duplicate.
UEditor for assistant markdown. Matches AIR-Bot. Template uses a custom ChatComark; cosmo already ships TipTap-via-UEditor, so no new dep.
Tools skipped (not 500'd) when no active org. createAITools skips registration during onboarding rather than throwing — keeps the chat surface usable for fresh users.
Old /api/app/ai/chat.post.ts deleted. Nothing referenced it post-cutover; leaving it would have invited drift between two streaming endpoints.

Golden path (what a fresh clone gets)

The cumulative shape of cosmo after this branch ships:

  1. Clone cosmo. Run npm install + cp .env.example .env + fill in SUPABASE_URL, SUPABASE_ANON_KEY, SUPABASE_SERVICE_ROLE_KEY, OPENAI_API_KEY. Replace {{PROJECT_REF}} in .cursor/mcp.json.
  2. Run migrations 001–008 against the new Supabase project.
  3. npm run dev — Nuxt + Inngest both boot. Visit /auth/signup, sign up.
  4. Get routed to /onboarding; create an org. Land on /app with the org context populated; TeamsMenu shows the org.
  5. Visit /app/billing — Free plan, usage bar, Pro upgrade card. "Upgrade" shows the demo-checkout banner with the stub toast.
  6. Visit /app/chat — empty state with greeting + suggestion grid. Submit a prompt; navigation animates to /app/chat/<id> with the prompt locked in place; assistant streams.
  7. Settings → Members → invite a teammate by email; Settings → Profile → set timezone + display name.
  8. Branded error page surfaces on createError. Help page reachable at /help. Analytics events flow to analytics.events.
  9. Set profiles.is_employee = true on a user → [Internal] nav group surfaces with /app/admin + /app/dev-tools. Non-employees still get a 404 on those routes.
  10. Flip to live Stripe by setting STRIPE_SECRET_KEY + price/webhook IDs and running stripe listen; nothing else changes.
